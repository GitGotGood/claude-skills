# Blindspot Audit Question Bank

Use these questions selectively. Do not dump them all into every audit.

## Frame
- What problem is this actually trying to solve?
- What problem is being assumed instead of proven?
- What is being treated as fixed that may not be fixed?
- What would make this an answer to the wrong question?

## Assumptions
- What must be true for this to work?
- Which assumption is carrying the most weight?
- Which assumption sounds obvious only because it has not been isolated?
- If one assumption were false, which one would collapse the plan fastest?

## Omissions
- Who is affected but not represented?
- What dependency is quietly doing most of the work?
- What cost is likely to appear only after initial success?
- What adjacent alternative never made it onto the table?

## Perspectives
- How would a competitor interpret this move?
- How would an operator feel the burden of this plan?
- What would a skeptical outsider say is over-assumed?
- What will future-you wish you had tested first?

## Metrics
- What is the actual true north here?
- What metric are we using because it is easy rather than right?
- How could this KPI rise while the real outcome deteriorates?
- What invisible variable matters but is not being measured?

## Premortem
- It failed in 12 months. Why?
- What warning sign would show up early?
- What success signal might be false comfort?
- What is the cheapest test that would meaningfully reduce uncertainty now?
